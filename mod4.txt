Deliverable #1: Four Database-Oriented Features
Feature 1 – Login and Rewards System (Tolu) Description: Extend the existing Marketplace
class to include a secure login feature that validates user credentials stored in a database. Once
logged in, users can earn and redeem rewards points.
Database Use: A Users table will store usernames, hashed passwords, and roles. A Rewards table
will store each user’s current points and redemption history.
Feature 2 – Product Expansion and Inventory Database (Jaimie) Description: Refactor the
current shoe-specific structure into a Product superclass that connects to a database of available
items.
Database Use: A Products table will hold product data (product ID, name, category, price,
quantity). The database will allow dynamic addition of categories beyond shoes, making the
store scalable.
Feature 3 – Order History Log (Tolu) Description: Add functionality for customers to view their
order history. Each order will be recorded in the database with details such as customer ID, date,
product list, and total cost.
Database Use: An Orders table will track purchases, connecting customer IDs to products
purchased, allowing customers and sellers to view previous transactions.
Feature 4 – Change Log for Sellers and Admins (Jaimie) Description: Implement a feature that
records updates to product listings (e.g., price or stock changes) into a change log.
Database Use: A ChangeLog table will store timestamps, product IDs, old and new values, and
the user who made the update. This helps with auditing and transparency in product
management.

Deliverable #3: Table-Feature Integration Summary
Users / Rewards → Login & Rewards System: These tables handle authentication and track
points earned for purchases.
Products → Product Expansion: Allows multiple product types and enables future scalability.
Orders / OrderDetails → Order History Log: Stores customer purchase records for easy retrieval
in user profiles.
ChangeLog → Seller/Admin Updates: Records product data changes to maintain version control
and accountability.
